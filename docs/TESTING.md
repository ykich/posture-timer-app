# PostureTimer テストガイド

## 概要

Swift 実装のテストは Xcode の XCTest フレームワークを使用する。

現時点では手動テストを中心に行う。自動テストの追加は今後の課題。

## 手動テスト項目

### 基本動作

| 操作 | 確認内容 |
|------|---------|
| アプリ起動 | メニューバーに `🪑 00:00` が表示される |
| 「立ち上がる」クリック | アイコンが `🧍` に変わり、経過時間がリセットされる |
| 「座る」クリック | アイコンが `🪑` に戻り、経過時間がリセットされる |
| 一時停止 | タイマーが止まり、メニューバーに `⏸` が表示される |
| 再開 | タイマーが再開し、一時停止中の時間が除外されている |
| 終了 | データが保存されてアプリが終了する |

### 通知

| 条件 | 確認内容 |
|------|---------|
| 初回起動 | 通知許可ダイアログが表示される |
| 座り警告時間を超過 | macOS バナー通知が届く |
| 立ち警告時間を超過 | macOS バナー通知が届く |
| 状態切り替え後 | 同じ状態で再度アラートが発火する |

### データ永続化

| 操作 | 確認内容 |
|------|---------|
| アプリ再起動 | 設定値（アラート時間・ホットキー）が引き継がれる |
| 設定変更後に再起動 | 警告時間の設定が保持されている |
